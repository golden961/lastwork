# Банкетам.Нет — Демонстрационный экзамен 2026 (Вариант 4)

Информационная система бронирования помещений для банкетов: зал / ресторан / летняя веранда / закрытая веранда.

**Специальность:** 09.02.07 «Информационные системы и программирование»  
**Квалификация:** Разработчик веб и мультимедийных приложений  
**ДЭ:** Базовый уровень • Вариант №4 • Шифр: В4_КОД 09.02.07-3-2026-БУ  
**Стек:** Vite + React (JS) + Node.js (backend) + PostgreSQL (Neon)

---

## 1) Функционал по ТЗ

### Пользователь
- Регистрация (обязательные поля): **логин, пароль, ФИО, телефон, e-mail**
    - Логин: **уникальный**, латиница+цифры, **≥ 6**
    - Пароль: **≥ 8**
    - Подсказки рядом с полями при ошибках
- Авторизация по логину/паролю + информативные уведомления
- Создание заявки на бронирование:
    - выбор помещения (4 типа)
    - дата начала банкета (формат **ДД.ММ.ГГГГ**)
    - способ оплаты (dropdown)
    - заявка сохраняется в БД со статусом **«Новая»**
- Личный кабинет:
    - история заявок
    - возможность оставить отзыв **только после изменения статуса администратором** (статус ≠ «Новая»)

### Администратор
- Вход: **Admin26 / Demo20**
- Просмотр всех заявок
- Изменение статуса заявки:
    - «Новая»
    - «Банкет назначен»
    - «Банкет завершен»
- Удобства в админке: фильтры, сортировка, пагинация, модальные окна, уведомления

### Дизайн (Модуль 2)
- Мобильный интерфейс: **390×844**
- Шрифт: **Oswald**
- Палитра по styleguide:
    - Gold `#DAA520`
    - Rose gold `#FFDAB9`
    - Cream `#FFFDD0`
    - Crimson `#DC143C`
    - Dark green `#006400`
- Слайдер (в ЛК):
    - 4 изображения одинакового размера
    - автопереключение каждые 3 секунды
    - кнопки вперед/назад
